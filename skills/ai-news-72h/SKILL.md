# 最近72小时 AI 新闻简报

## 目标

生成一份可追溯、结构化的中文 AI 新闻简报。严格区分：

- 事件发生时间与文章发布时间；
- 正式发布、预告、传闻和评论；
- 企业自述与独立第三方评价；
- 本时间窗内的新进展与仅用于解释背景的旧资料。

不得为了凑数量纳入窗口外旧闻，或把推测写成已经发生的事实。

## 隔离执行

每次运行本 Skill 必须由一个全新 subagent 独立完成检索、筛选、验证和 Markdown 文件写入，避免继承当前会话中的候选新闻、判断或结论。

使用以下无递归协议：

1. 如果任务输入中没有 `execution_role: isolated_worker`，当前 agent 是协调者：
   - 创建一个 fresh subagent，设置 `fork_context=false`；
   - 仅传递当前用户请求、当前绝对时间与时区、输出目录、执行模型标识，以及本 Skill 和所需 reference 文件；
   - 在任务中加入 `execution_role: isolated_worker`；
   - 不向 subagent 传递此前对话、既有候选条目、旧简报或当前 agent 的研究判断；
   - 不在当前 agent 中重复执行新闻检索，仅在 subagent 完成后检查输出文件是否存在、命名是否正确、验证清单是否完成。
2. 如果任务输入中已经包含 `execution_role: isolated_worker`，当前 agent 是隔离执行者：直接执行完整工作流，不再创建嵌套 subagent。
3. 如果运行环境不支持 subagent，明确报告无法满足隔离执行要求，不得静默回退到当前会话直接运行。

## 输出文件命名

最终结果必须写入 Markdown 文件。文件名在 `.md` 前加入实际用于生成该文件的 AI 模型名称：

- 用户指定基础文件名时：`<基础文件名>_<model-name>.md`；
- 用户未指定时：`ai-news-72h_<YYYY-MM-DD>_<model-name>.md`；
- 示例：`ai-news-72h_2026-07-01_deepseek-V4-pro.md`。

`model-name` 必须使用隔离执行 subagent 的实际模型标识，不得使用协调者模型名，也不得猜测营销名称。将空格、斜杠、反斜杠及文件系统不允许的字符替换为连字符，保留字母、数字、点、下划线和连字符。输出正文开头同时记录 `**生成模型：<model-name>**`。

## 工作流

### 1. 确定时间窗

1. 获取当前日期、时间和用户时区；未指定时使用会话提供的时区。
2. 计算精确的72小时起止时间，并在正文开头明确写出。
3. 对每条候选新闻记录：
   - 事件发生时间；
   - 首次公开时间；
   - 当前来源发布时间及其时区。
4. 只有事件或首次正式公开发生在窗口内，才能列为本期新闻。
5. 若来源只显示日期而没有时间，在文中标注“按发布日期计入”。

### 2. 分轨检索

至少检索下列类别，不要求每类都必须有新闻：

- 模型：新模型、推理、多模态、开源模型、API、价格与上下文能力。
- Agent：编码 Agent、浏览器 Agent、企业工作流、协议和工具调用。
- 硬件：GPU、ASIC、服务器、边缘设备、机器人和 Physical AI。
- 软件与平台：开发工具、云服务、操作系统、数据平台和安全工具。
- 应用与产业：医疗、教育、金融、内容、企业部署、融资与监管影响。
- AI 科学研究：OpenAI、Anthropic、Google Research/Google DeepMind，以及其他重要实验室的 AI for Science 工作。

使用多个不同关键词和来源检索，避免只围绕已知公司搜索。对“最近”“今天”等查询加入绝对日期。

### 3. 建立证据链

每条新闻优先收集：

1. 第一方原始来源：公司公告、研究博客、论文、代码库、监管文件或产品文档。
2. 至少一个独立来源：Reuters、AP、Financial Times、Bloomberg、主流科技媒体、学术机构评论或领域专家分析。
3. 必要时加入第二个独立来源，处理争议、商业影响或技术能力主张。

技术问题优先使用官方文档和原始论文。不要仅根据搜索结果摘要下结论。

AI for Science 预印本的证据要求见 [references/ai-for-science-sources.md](references/ai-for-science-sources.md)。窗口内首次发布的预印本可仅依据原始论文页面纳入，但必须执行该文件中的状态标注和验证规则。

若只有企业公告：

- 可以报道，但明确标注“目前主要依据公司公告”；
- 第三方重要性评价必须由作者独立分析，不得伪装成外部共识；
- 降低结论确定性。

### 4. 筛选与排序

对候选新闻按以下维度评分：

- 新颖性：是否在72小时内首次公开；
- 影响范围：是否影响开发者、企业、消费者或科研群体；
- 技术实质：是否有模型、产品、论文、代码或可验证能力；
- 来源质量：是否有第一方材料和独立交叉验证；
- 持续影响：是否可能改变成本、竞争格局、工作流或研究方法。

优先保留高影响、高证据质量新闻；宁缺毋滥。

### 5. AI 科学研究专章

单列 `AI 科学研究` 章节。执行任何 AI for Science 检索前，必须完整读取并遵循 [references/ai-for-science-sources.md](references/ai-for-science-sources.md)，其中规定：

- 必查预印本、重点生命医学期刊和实验室官方渠道；
- 各来源的72小时时间认定方式；
- AI 核心贡献的纳入与排除标准；
- 预印本状态、消息类型和研究条目字段；
- AI for Science 专项输出结构和验证清单。

核心约束不得省略：只关注首次公开内容；arXiv 按 `v1` 判断；不纳入后续版本、旧论文新公告或 PubMed 结果；没有符合条件的新研究时如实报告。

### 6. 撰写重要性评价

每条新闻加入独立的“第三方重要性评价”，评价应：

- 说明为何重要、影响谁、时间尺度多长；
- 区分事实、来源观点和作者推断；
- 同时指出至少一个限制、风险或尚待验证的问题；
- 使用 `高 / 中高 / 中 / 低` 等级，并给出理由；
- 避免把融资额、参数量、基准分数直接等同于实际影响力。

若评价没有引用具体第三方，应写成“综合评价”或“本简报判断”，不得称为第三方共识。

## 输出结构

按以下顺序输出：

```markdown
# 最近72小时全球 AI 新闻简报

**统计窗口：YYYY-MM-DD HH:mm 至 YYYY-MM-DD HH:mm（时区）**
**检索截止：YYYY-MM-DD HH:mm（时区）**
**生成模型：<model-name>**

## Introduction
概括本期最重要的2至4条趋势，并说明是否存在重大的模型发布。

## 一、[类别]

### [具体新闻标题]
**事件日期：**
**状态：正式发布 / 预告 / 媒体报道 / 尚待确认**

#### 摘要
说明发生了什么、涉及谁、关键数字和能力边界。

#### 第三方重要性评价
**重要性：高 / 中高 / 中 / 低**
给出影响、限制和独立判断，明确推断性质。

#### 来源
- [第一方来源标题](URL)
- [独立来源标题](URL)

## AI 科学研究
按 [references/ai-for-science-sources.md](references/ai-for-science-sources.md) 中的专项结构撰写早期研究、重点期刊、实验室科研动态和本期总结。

## 全文总结
提炼产业、技术、应用和科学研究的共同趋势，避免逐条重复摘要。

## 方法与局限
说明时间窗、检索截止时间、来源限制和仍待确认的事项。
```

标题可按实际内容设置为模型、Agent、硬件、软件、应用、产业或监管，不要为没有可靠新闻的类别制造占位内容。

## 写作规则

- 使用中文，保留必要的英文产品名、机构名和论文名。
- 每条新闻只讲一个核心事件。
- 给出具体绝对日期，不使用含糊的“昨天”“近日”。
- 摘要以事实为主，不重复公司营销措辞。
- 所有关键数字、发布日期和能力声明都应有来源。
- 链接尽量指向原始页面，不使用搜索结果页或聚合转载页。
- 论文必须标注正式发表或预印本状态。
- 预印本必须标注首次版本；arXiv 明确记录 `v1` 时间。
- 对付费墙来源，可引用标题和可核验事实，但应同时寻找公开来源。
- 不把新闻文章的发表日期误当作事件发生日期。

## 验证检查

提交前逐项检查；任何关键项失败都应继续检索或删除相关条目。

### 时间验证

- [ ] 已写明精确的72小时起止时间、时区和检索截止时间。
- [ ] 每条新闻的事件或首次公开时间确实位于窗口内。
- [ ] 窗口外资料仅作为背景，并明确标注原始日期。
- [ ] 没有把未来预告写成已经发布。
- [ ] AI for Science 内容已完成引用文件中的专项时间验证。

### 事实与来源验证

- [ ] 每条新闻至少有一个可打开的原始或高质量来源。
- [ ] 重大新闻优先同时具备第一方来源和独立来源。
- [ ] 标题、日期、公司、产品名、数字和链接逐项核对。
- [ ] 搜索摘要未被当作唯一证据。
- [ ] 传闻、匿名消息和未经验证的主张已显著标注。
- [ ] 没有虚构新闻、引文、论文、链接或第三方观点。
- [ ] AI for Science 内容已完成引用文件中的专项来源验证。

### 内容验证

- [ ] Introduction、新闻标题、摘要、重要性评价、来源和全文总结齐全。
- [ ] AI 科学研究为独立章节。
- [ ] AI for Science 内容已完成引用文件中的专项内容验证。
- [ ] 没有新研究时如实报告，而非拿旧研究凑数。
- [ ] 每项重要性评价同时包含影响理由和限制条件。
- [ ] “第三方评价”有真实第三方依据；否则已改称“综合评价”。

### 质量验证

- [ ] 任务由带有 `execution_role: isolated_worker` 的 fresh subagent 独立执行。
- [ ] subagent 未继承或接收此前对话中的候选新闻与研究判断。
- [ ] 输出已写入 Markdown 文件，文件名以实际执行模型名称结尾。
- [ ] 正文开头记录的生成模型与文件名中的模型名称一致。

- [ ] 各类别覆盖由真实新闻决定，没有机械凑齐。
- [ ] 事实、来源观点和本简报推断表述清楚。
- [ ] 全文总结提炼趋势，而非复制各条摘要。
- [ ] 所有链接格式正确，并尽可能打开抽查。

最后在内部做一次反向核验：尝试证明每条新闻不应被纳入。无法通过时间、来源或事实核验的条目，删除或移入明确标注日期的背景部分。
